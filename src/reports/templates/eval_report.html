2.1 Используемые метрики

Для оценки качества RAG системы использованы следующие метрики DeepEval:

Answer Relevancy (AR)

Оценивает соответствие ответа заданному вопросу. Высокий показатель указывает на то, что ответ напрямую отвечает на поставленный вопрос.

Faithfulness (FA)

Оценивает соответствие ответа предоставленному контексту (retrieved chunks). Показывает, насколько ответ основан на фактах из документов, а не на галлюцинациях модели.

  • Порог: {{ (fa_threshold * 100) | round(0) }}%
  • Среднее значение: {{ (fa_avg * 100) | round(1) }}%
  • Pass Rate: {{ (fa_pass_rate * 100) | round(1) }}%
{% endif %} {% if cp_avg is not none %}

Contextual Precision (CP)

Оценивает точность извлеченного контекста относительно эталонного ответа. Высокий показатель означает, что релевантные чанки находятся в начале списка.

  • Порог: {{ (cp_threshold * 100) | round(0) }}%
  • Среднее значение: {{ (cp_avg * 100) | round(1) }}%
  • Pass Rate: {{ (cp_pass_rate * 100) | round(1) }}%
{% endif %} {% if cr_avg is not none %}

Contextual Recall (CR)

Оценивает полноту извлеченного контекста. Показывает, насколько хорошо система нашла все необходимые чанки для ответа на вопрос.

5. Рекомендации

{% if ar_avg < ar_threshold %}

🔴 Answer Relevancy ниже порога

Проблема: Ответы системы недостаточно релевантны заданным вопросам ({{ (ar_avg * 100) | round(1) }}% < {{ (ar_threshold * 100) | round(0) }}%).

Рекомендации:

  • Улучшить промпт для генерации ответов
  • Убедиться, что модель понимает контекст вопроса
  • Проверить качество retrieval — возможно, извлекаются нерелевантные чанки
{% endif %} {% if fa_avg is not none and fa_avg < fa_threshold %}

🔴 Faithfulness ниже порога

Проблема: Система генерирует ответы, не основанные на предоставленном контексте (галлюцинации).

Рекомендации:

  • Добавить в промпт explicit instruction опираться только на предоставленный контекст
  • Использовать модель с меньшей склонностью к галлюцинациям
  • Добавить постобработку для фильтрации некорректных ответов
{% endif %} {% if cp_avg is not none and cp_avg < cp_threshold %}

🟡 Contextual Precision требует улучшения

Проблема: Релевантные чанки находятся не в начале списка retrieved context.

Рекомендации:

  • Улучшить алгоритм ранжирования (reranker)
  • Настроить embedding модель под специфику задачи (fine-tuning)
  • Проверить метрики similarity — возможно, threshold слишком низкий
{% endif %} {% if cr_avg is not none and cr_avg < cr_threshold %}

🟡 Contextual Recall требует улучшения

Проблема: Система не находит все необходимые чанки для ответа.

Рекомендации:

  • Увеличить значение top_k при retrieval
  • Проверить chunking strategy — возможно, релевантная информация разбита между несколькими чанками
  • Рассмотреть использование hybrid search (BM25 + vector)
{% endif %} {% if ar_avg >= ar_threshold and (fa_avg is none or fa_avg >= fa_threshold) %}

{% endif %}
