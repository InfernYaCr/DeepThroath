# DeepThroath — Product Concept

## Vision

**DeepThroath** превращает разовый аудит безопасности LLM в непрерывный процесс мониторинга.

Платформа закрывает разрыв между командами ML-инженеров, создающих AI-продукты, и требованиями безопасности, compliance-отделов и клиентов. Каждое изменение системного промпта или весов модели автоматически проходит красный тест и порождает измеримый отчёт с конкретными доказательствами.

---

## Проблема

Компании, разрабатывающие продукты на основе LLM, сталкиваются с несколькими болями:

1. **Нет объективной метрики безопасности.** "Мы тестировали вручную" — не аргумент для корпоративного клиента и не соответствует регуляторным требованиям.

2. **Ручное red teaming дорого и нерегулярно.** Найм специалистов по безопасности AI стоит $200–500/час и даёт снимок на момент аудита, а не непрерывный контроль.

3. **Нет трассируемости.** После изменения промпта команда не знает, стала ли модель безопаснее или уязвимее по конкретным OWASP категориям.

4. **Клиентам нечего показать.** Корпоративные покупатели всё чаще требуют доказательства безопасности AI-систем в формате стандартизированных отчётов.

---

## Решение

DeepThroath предоставляет:

| Возможность | Ценность |
|-------------|----------|
| Автоматизированные атаки через DeepTeam | Сотни тестов за минуты вместо дней ручной работы |
| OWASP LLM Top 10 классификация | Стандартизированный язык общения с клиентом и аудиторами |
| Security Score (0–100) | Один понятный показатель для руководства |
| Trend-анализ по версиям | Видно, улучшилась ли безопасность после каждого деплоя |
| Клиентский PDF/HTML отчёт | Готовый документ для корпоративных продаж и compliance |
| CI/CD интеграция | Автоматический блок деплоя при превышении порога ASR |

## Целевые пользователи

### Основная аудитория

**AI Product Teams** в компаниях, которые:
- Встраивают LLM в корпоративные продукты (чат-боты, ассистенты, аналитика)
- Работают в регулируемых отраслях: финтех, healthtech, legalttech, HR
- Продают B2B, где безопасность — часть RFP/RFI или условие сделки

**Роли:**
- ML Engineers — интеграция в CI/CD, настройка атак
- Security Engineers — анализ результатов, закрытие уязвимостей
- Product Managers — Security Score как KPI продукта
- Pre-sales / Solutions Engineers — генерация отчётов для клиентов

### Вторичная аудитория

- **Independent Security Researchers** — аудит чужих LLM-систем
- **Compliance Officers** — документирование соответствия AI Act, NIST AI RMF
- **AI Consultancies** — инструмент для платных аудитов

---

## Ключевые метрики продукта

| Метрика | Описание |
|---------|----------|
| Security Score | Взвешенный pass_rate (0–100), понятен нетехническим стейкхолдерам |
| ASR per OWASP Category | Детальный разрез по 10 категориям угроз |
| Guardrail Bypass Rate | Специфично для продуктов с фильтрами контента |
| Cross-version Delta | Изменение Security Score между релизами |
| Time-to-detect | Как быстро CI/CD обнаружил регрессию безопасности |

## Конкурентная карта

| Продукт | Тип | Ограничения |
|---------|-----|-------------|
| Garak (NVIDIA) | Open-source CLI | Нет дашборда, нет клиентских отчётов |
| PromptFoo | Open-source | Фокус на качестве, не на безопасности |
| Lakera Guard | Runtime фильтр | Не тестирует, только блокирует |
| Adversa AI | Enterprise | Высокая стоимость, длинный цикл продаж |
| Ручной red team | Услуга | Дорого, нерегулярно, не масштабируется |
| **DeepThroath** | **Platform** | **Автоматизация + отчётность + CI/CD** |

## Сценарии использования

### Сценарий 1 — Релизный gates
Команда изменила системный промпт AI-ассистента. PR в GitHub запускает DeepThroath. Если ASR > 20% — PR не мёрджится. Через 10 минут в PR появляется комментарий с Security Score и списком новых уязвимостей.

### Сценарий 2 — Корпоративная продажа
Клиент (банк) спрашивает: "Докажите, что ваш AI-ассистент защищён от prompt injection." Вместо устных заверений — автоматически сгенерированный PDF-отчёт с результатами 150 тестов, Security Score 87/100, и конкретными примерами отражённых атак.

### Сценарий 3 — Регуляторный аудит
Compliance-команда должна подготовить документацию для EU AI Act. DeepThroath предоставляет хронологию Security Score по всем версиям модели за квартал, с привязкой к OWASP LLM Top 10.

### Сценарий 4 — Сравнение провайдеров
Компания выбирает между GPT-4o и Claude 3.5 для встройки в продукт. DeepThroath прогоняет одинаковый набор атак на оба и возвращает сравнительный отчёт.

## Roadmap

### v1.0 (текущий)
- Интеграция DeepTeam
- 6 категорий уязвимостей, 3 типа атак
- Streamlit дашборд
- PDF/HTML клиентский отчёт
- GitHub Actions CI/CD

### v1.1
- Поддержка OpenAI, Mistral, локальных моделей (Ollama)
- Slack/Teams уведомления о деградации Security Score
- Кастомные уязвимости через YAML

### v2.0
- Multi-tenant SaaS (web UI без локальной установки)
- API для интеграции в любой CI/CD
- Сравнение нескольких моделей в одном отчёте
- Исторический бенчмарк относительно индустрии

### v3.0
- Continuous monitoring (не только при деплое, но и в продакшне)
- Автоматические рекомендации по исправлению промптов
- Интеграция с Jira / Linear для создания тикетов по уязвимостям
